# Phase 1 -- UI Design Contract

> Visual and interaction contract for Foundation and Auth. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (to be initialized during scaffolding) |
| Preset | Custom monochrome dark -- no preset file, manual CSS variable override in globals.css |
| Component library | 8bitcn/ui (built on Radix primitives via shadcn) |
| Icon library | None required for Phase 1 (no icons in auth forms or nav beyond text) |
| Heading font | Press Start 2P (Google Fonts via next/font/google, variable: --font-press-start) |
| Body font | Geist (via next/font, variable: --font-geist) |
| Monospace font | Geist Mono (via next/font, variable: --font-geist-mono) |

**Note for executor:** 8bitcn's `retro.css` imports Press Start 2P via `@import url(...)` from Google Fonts. Override this by ensuring the `next/font/google` CSS variable (`--font-press-start`) is applied to `.retro` class in globals.css, or remove the `@import` line from the copied `retro.css` file to avoid double-loading and CLS.

## Spacing Scale

Declared values (multiples of 4, Tailwind defaults):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1, gap-1) | Icon gaps, inline padding within buttons |
| sm | 8px (p-2, gap-2) | Compact element spacing, input internal padding |
| md | 16px (p-4, gap-4) | Default element spacing, form field gaps |
| lg | 24px (p-6, gap-6) | Card padding, section padding within auth forms |
| xl | 32px (p-8, gap-8) | Layout gaps between major form sections |
| 2xl | 48px (p-12, gap-12) | Page-level vertical padding (auth page top/bottom) |
| 3xl | 64px (p-16, gap-16) | Major page section breaks (not used in Phase 1) |

Exceptions: None for Phase 1. All spacing follows the 4/8 base grid.

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Geist | 16px (text-base) | 400 (regular) | 1.5 | Form labels, descriptions, body text, error messages |
| Small | Geist | 14px (text-sm) | 400 (regular) | 1.5 | Helper text below inputs, legal footnotes |
| Heading | Press Start 2P | 20px (text-xl) | 400 (only weight) | 1.4 | Page titles ("Sign Up", "Log In", "Reset Password") |
| Display | Press Start 2P | 28px (text-2xl) | 400 (only weight) | 1.2 | App name in header, hero text on home placeholder |

**Constraints:**
- Press Start 2P is never used below 14px -- it becomes illegible. At 14px, use sparingly (nav links only).
- All body text, form inputs, and button labels use Geist at 16px minimum.
- Press Start 2P only has weight 400. Do not attempt bold/semibold.
- Geist uses weight 400 (regular) for body and weight 600 (semibold) for emphasis (button labels, strong text).

## Color

All colors defined as HSL CSS variables (shadcn convention). This is a monochrome-only palette -- no hue, no saturation except for destructive red.

| Role | HSL Value | Hex Equivalent | Usage |
|------|-----------|----------------|-------|
| Dominant (60%) -- background | 0 0% 0% | #000000 | Page background, canvas background |
| Secondary (30%) -- card/chrome | 0 0% 4% | #0a0a0a | Card backgrounds, form containers, header background |
| Secondary alt -- elevated surface | 0 0% 7% | #111111 | Input backgrounds, secondary surfaces |
| Foreground -- primary text | 0 0% 100% | #ffffff | All text, borders, pixelated UI elements |
| Muted -- subdued surface | 0 0% 10% | #1a1a1a | Hover states, disabled backgrounds |
| Muted foreground -- secondary text | 0 0% 64% | #a3a3a3 | Placeholder text, helper text, timestamps |
| Border | 0 0% 15% | #262626 | Input borders, card borders, dividers |
| Ring -- focus indicator | 0 0% 100% | #ffffff | Focus rings on inputs and buttons |
| Destructive | 0 62% 30% | #7c1d1d | Error states, destructive actions only |
| Destructive foreground | 0 0% 100% | #ffffff | Text on destructive elements |

**Accent reserved for:** In this monochrome design, "accent" is white (#ffffff) on black. White is reserved for: primary CTA buttons (filled), active navigation state, focus rings, and the dot motif. There is no colored accent in Phase 1.

**Border radius:** 0px globally (`--radius: 0`). The 8-bit aesthetic requires sharp corners on all elements. No rounded corners anywhere.

## Interaction States

### Auth Form Behavior

| State | Visual Treatment |
|-------|------------------|
| Input default | Border: #262626, background: #111111, text: #ffffff |
| Input focused | Ring: 2px solid #ffffff (focus-visible only), border: #ffffff |
| Input error | Border: #7c1d1d (destructive), error text below in destructive color at 14px |
| Input disabled | Opacity 50%, cursor not-allowed |
| Button default | White background, black text, pixelated 8-bit border (from 8bitcn) |
| Button hover | No color change (monochrome), slight opacity shift or invert handled by 8bitcn |
| Button loading | Button text replaced with "..." or pulsing dot, disabled state |
| Button pressed | translate-y-1 (built into 8bitcn button) |

### Navigation Header

| State | Visual Treatment |
|-------|------------------|
| Logged out | Show "Log In" and "Sign Up" as text links in Press Start 2P at 14px |
| Logged in | Show username (Geist, 14px) and "Log Out" button (8bitcn button, outline variant) |

### Form Validation

- **Client-side first:** Zod validation runs on blur and on submit. Errors display immediately below the field.
- **Server-side second:** Server Action returns errors. Map to field-level display.
- **Username availability:** Check on blur after format validation passes. Show inline "available" or "taken" indicator.

## Layout Contract

### Auth Pages Layout

```
+------------------------------------------+
|  [Header: Logo/Title ... Nav Links]       |  h: 64px, bg: #0a0a0a
+------------------------------------------+
|                                          |
|          +------------------+            |  centered vertically + horizontally
|          |   Card (480px)   |            |  max-width: 480px
|          |                  |            |  padding: 24px (lg)
|          |  [Heading]       |            |  Press Start 2P, 20px
|          |  [Subtext]       |            |  Geist, 14px, muted-foreground
|          |                  |            |
|          |  [Form Fields]   |            |  gap: 16px between fields
|          |  [CTA Button]    |            |  full-width button
|          |                  |            |
|          |  [Alt Link]      |            |  "Already have an account? Log In"
|          +------------------+            |
|                                          |
+------------------------------------------+
```

- Auth card is vertically and horizontally centered on the viewport.
- Card max-width: 480px. On mobile (<640px), card is full-width with 16px horizontal margin.
- Card uses `bg-card` (#0a0a0a) with 8bitcn pixelated border treatment.
- Alt links below forms (e.g., "Already have an account? Log In") use Geist at 14px, muted-foreground color, with white on hover.
