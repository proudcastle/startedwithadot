2. Create `app/icon.tsx` per D-08, D-14 (SEO-02). This is a Next.js file-convention route that generates the favicon dynamically using ImageResponse from "next/og".

Export constants: `size` = { width: 32, height: 32 }, `contentType` = "image/png".

Default async export function `Icon()` returns `new ImageResponse(jsx, { ...size })`. The JSX:
- Outer div: black background (#000), 100% width/height, display flex, alignItems center, justifyContent center
- White dot: div with width 20, height 20, borderRadius "50%", background "white"

That is the entire component. Next.js automatically generates the favicon link tag from this file. The 32x32 size covers the standard favicon use case. Browsers downscale to 16x16 as needed.

No need for a separate favicon.ico when icon.tsx exists -- Next.js handles the routing. If the build warns about missing favicon.ico, create a minimal one, but typically icon.tsx is sufficient.
