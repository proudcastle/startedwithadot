## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, dot separator dot spacing |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card padding, section inner gaps |
| lg | 24px | Section padding, footer inner padding |
| xl | 32px | Layout gaps between landing page subsections |
| 2xl | 48px | Major section breaks (between landing page sections) |
| 3xl | 64px | Page-level top/bottom padding, hero vertical padding |

Exceptions: none

Source: Standard 8-point scale. Existing codebase uses Tailwind spacing utilities (`px-5`, `gap-4`, `py-8`) which map to this scale.

## Typography

| Role | Font | Size | Weight | Line Height |
|------|------|------|--------|-------------|
| Body | Geist (--font-sans) | 16px (text-base) | 400 (regular) | 1.5 |
| Label / Small | Geist (--font-sans) | 14px (text-sm) | 400 (regular) | 1.5 |
| Heading | Press Start 2P (--font-press-start) | 14px (text-sm) | 400 (only weight available) | 1.4 |
| Display | Press Start 2P (--font-press-start) | 20px (text-xl) | 400 (only weight available) | 1.3 |

Notes:
- Press Start 2P only ships weight 400. Do not attempt bold/semibold.
- Press Start 2P is illegible below 10px. The 14px heading size is the floor for this font.
- Display size (20px) is used for landing page hero text and section titles only.
- Geist weight 400 for body, 600 (semibold) for emphasis (username, CTA labels, nav links).
- Existing pattern: `font-[family-name:var(--font-press-start-2p)]` for pixel headings, `font-sans` for body.

Source: `app/layout.tsx` (font loading), `components/header.tsx` (existing usage patterns), CLAUDE.md constraints.

## Color

| Role | CSS Variable | HSL Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | --background | 0 0% 0% (pure black) | Page background, canvas background, hero background |
| Secondary (30%) | --card / --secondary | 0 0% 4% / 0 0% 7% | Cards, header, footer, proposal cards, version cards |
| Accent (10%) | --primary | 0 0% 100% (pure white) | See reserved list below |
| Muted | --muted-foreground | 0 0% 64% | Timestamps, secondary labels, dot separators, placeholder text |
| Destructive | --destructive | 0 62% 30% | Delete proposal confirmation only |
| Border | --border | 0 0% 15% | Card borders, header border, input borders |

Accent (white, --primary) reserved for:
1. Primary CTA buttons (filled white with black text)
2. The pulsing dot on canvas
3. The dot favicon
4. Active nav link text
5. DotLoader pulsing dot
6. Vote dot counters when user has voted
7. Focus ring (--ring)

Everything else uses --muted-foreground (gray 64%) or --foreground (white at reduced opacity via text classes).

Source: `app/globals.css` (full theme definition), CLAUDE.md constraint "strictly monochrome."

## Interaction Patterns

| Interaction | Behavior |
|-------------|----------|
| Landing page scroll | Vertical scroll through sections, no parallax, no scroll hijacking |
| CTA "Sign up" link | Navigates to `/auth/sign-up` |
| CTA "See all proposals" link | Navigates to `/proposals` |
| CTA "Full changelog" link | Navigates to `/changelog` |
| 404 "Back to the dot" CTA | Navigates to `/` |
| DotLoader | CSS pulse animation (scale 0.8 to 1.2, 1.5s cycle, infinite, ease-in-out) |
| DotSeparator | Static, no animation, no interaction |
| Footer links | Standard navigation, no special behavior |
| Responsive breakpoints | Mobile-first, existing Tailwind breakpoints (sm:640, md:768, lg:1024) |

No new interactive states introduced in this phase. All interactivity (voting, proposals, auth) was built in prior phases.
