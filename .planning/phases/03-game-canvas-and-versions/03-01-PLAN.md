    Constants at module level: BASE_RADIUS = 20, PULSE_AMOUNT = 2, CYCLE_MS = 3000.

    The component exports a named function GameCanvas with no props.

    Refs: canvasRef (HTMLCanvasElement), frameRef (number for RAF ID), sizeRef ({width, height} for current container dimensions). All mutable animation state in useRef, NOT useState per D-03/GAME-05.

    scaleCanvas callback (wrapped in useCallback, empty deps): reads canvasRef.current.parentElement getBoundingClientRect, sets sizeRef.current, applies devicePixelRatio scaling by multiplying canvas.width and canvas.height by dpr, then sets canvas.style.width/height to CSS pixel values, and calls ctx.scale(dpr, dpr). This handles HiDPI sharpness per GAME-03.

    First useEffect (deps: [scaleCanvas]): gets canvas context, calls scaleCanvas(), defines draw(timestamp) function that computes radius via pure sine wave (BASE_RADIUS + PULSE_AMOUNT * Math.sin(timestamp / CYCLE_MS * 2 * Math.PI)), clears canvas with #000000 fillRect, draws white (#FFFFFF) circle at center (sizeRef.current.width/2, sizeRef.current.height/2) using ctx.arc. Inside draw, reset transform with ctx.setTransform(dpr, 0, 0, dpr, 0, 0) before drawing to handle resize mid-frame. Kicks off RAF loop, cleanup returns cancelAnimationFrame(frameRef.current).

    Second useEffect (deps: [scaleCanvas]): creates ResizeObserver on canvas.parentElement, observer callback calls scaleCanvas(). Cleanup disconnects observer. Per RESEARCH.md Pitfall 3, only update canvas dimensions in the observer -- do not change container layout.

    JSX return: outer div with className "w-full" and style={{ minHeight: "60vh" }} as the container. Canvas element inside with ref={canvasRef}, className="block w-full h-full", role="img", aria-label="A pulsing white dot on a black background" per UI-SPEC accessibility notes.
