**Accent (white) reserved for these specific elements only:**
1. The dot itself on the canvas (white circle, `#FFFFFF`)
2. Version number link text in header ("v0.1 ●")
3. Version card title text (Press Start 2P headings)
4. The dot symbol ("●") used as visual connector between version number and title

**Canvas-specific colors (not CSS variables -- hardcoded in canvas):**
- Canvas fill: `#000000` (pure black, matches `--background`)
- Dot fill: `#FFFFFF` (pure white)
- No gradients, no shadows, no opacity variations on the canvas itself

## Interaction Contract

### GameCanvas Interactions

| Interaction | Behavior |
|-------------|----------|
| Page load | Canvas initializes, dot appears at center, pulse animation begins immediately |
| Browser resize | ResizeObserver triggers canvas resize + redraw. Dot stays centered. No flicker. |
| Tab hidden | RAF loop continues (browser throttles automatically). No manual visibility API handling in V1. |
| Navigation away | useEffect cleanup cancels RAF via `cancelAnimationFrame(frameId.current)`. No memory leak. |
| HiDPI display | Canvas scaled by `devicePixelRatio` for sharp rendering. CSS size stays at container dimensions. |

### Canvas Animation Spec

| Property | Value |
|----------|-------|
| Base radius | 20px (in CSS pixels, scaled by DPR on canvas) |
| Pulse range | 18px to 22px (plus/minus 2px) |
| Cycle duration | 3000ms (3 seconds full cycle) |
| Easing | Pure sine wave: `radius = 20 + 2 * Math.sin(elapsed * 2 * Math.PI / 3000)` |
| Frame rate | Native RAF (~60fps, browser-controlled) |
| State storage | `useRef` for animationFrameId, startTime. Zero `useState` calls for animation. |

### Changelog Interactions

| Interaction | Behavior |
|-------------|----------|
| Page load | Server Component renders all versions. No client JS required. |
| No versions | Empty state with narrator copy displayed (see Copywriting Contract) |
| Version badge click (header) | Navigates to `/changelog` via standard Next.js Link |

## Copywriting Contract

| Element | Copy | Voice |
|---------|------|-------|
| Changelog page heading | "The Evolution" | Narrator (retro, dramatic) |
| Changelog page subheading | "Every version, decided by you." | Narrator (inclusive) |
| Empty state heading | "No versions yet." | Narrator (matter-of-fact) |
| Empty state body | "The dot is waiting for its first evolution. Go propose something." | Narrator (prodding, with CTA) |
| Empty state CTA link | "Propose an idea" | Links to `/proposals` |
| Version zero display | "v0 ●" | Displayed in header when no versions exist |
| Version card date prefix | None -- use `timeAgo()` output directly (e.g., "3 days ago") | Functional |
| Version card author prefix | "@" before username (e.g., "@creator") | Matches proposal card pattern |
| Version card proposal label | "Proposed:" followed by truncated proposal text (max 60 chars + ellipsis) | Functional, contextual |

No primary CTA button exists in this phase -- the canvas and changelog are passive viewing experiences. The empty state "Propose an idea" link is the closest to a CTA.

No destructive actions exist in this phase. Version creation and proposal management were handled in Phase 2.

No error states specific to this phase -- the canvas renders client-side with no data fetching, and the changelog is a simple server-rendered query. If the Supabase query fails, Next.js error boundary handles it (established in Phase 1).

## Accessibility Notes

| Element | Requirement |
|---------|------------|
| Canvas | `role="img"` with `aria-label="A pulsing white dot on a black background"`. Canvas content is purely decorative -- screen readers get the label, not the animation. |
| Version badge (header) | Rendered as `<Link>` with visible text "v0.1 ●" -- inherently accessible. |
| Changelog cards | Semantic HTML: version title in `<h2>` or `<h3>`, description in `<p>`, metadata in `<time>` with `datetime` attribute. |
| Empty state | CTA link ("Propose an idea") must be a visible, focusable `<Link>` element, not just styled text. |
